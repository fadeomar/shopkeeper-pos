5. POS quantity can become NaN

If the cashier clears the number input, Number("") becomes 0, but other invalid cases can still create bad values. If NaN enters draft state, totals can break.

Fix:

function updateQuantity(productId: string, quantity: number) {
  const safeQuantity = Number.isFinite(quantity)
    ? Math.trunc(quantity)
    : 1;

  setDraftItems((cur) =>
    cur.map((i) =>
      i.productId === productId
        ? {
            ...i,
            quantity: Math.max(1, Math.min(safeQuantity, i.availableStock)),

        : i,
    ),
  );
}

4. Return quantity should also require whole numbers in service layer

Should be:
